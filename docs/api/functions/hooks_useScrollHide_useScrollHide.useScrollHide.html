    Function useScrollHide

    • useScrollHide(threshold?: number, hysteresis?: number): UseScrollHideReturn

      Custom hook to manage the visibility of an element based on scroll direction.

      Parameters

      • Optionalthreshold: number = 10

        The distance from the top of the page (in pixels) where the element is always visible.

      • Optionalhysteresis: number = 20

        The minimum distance (in pixels) that must be scrolled in a new direction before visibility changes.

      Returns UseScrollHideReturn

      State and functions for visibility control.

      Remarks

      Typically used for toolbars or headers, it hides the element when scrolling down and reveals it when scrolling up. It includes hysteresis to prevent flickering and a "force show" mechanism for programmatic control.

      It uses requestAnimationFrame for performance and handles edge cases like iOS scroll bounce and bottom-of-page triggers.

      See

      • import('../useScrollGridIntoView/useScrollGridIntoView').useScrollGridIntoView for coordination with programmatic scrolling.
      • import('../useScrollGridIntoView/useScrollGridIntoView').registerToolbarForceShow for the global callback registration.

      Hook

      Example: Visibility management in a component

      const MyToolbar = () => {
        const { isVisible, toolbarRef, forceShow } = useScrollHide(50, 20);

        return (
          <header
            ref={toolbarRef}
            style={{
              transform: isVisible ? 'translateY(0)' : 'translateY(-100%)',
              transition: 'transform 0.3s ease-in-out'
            }}
          >
            Sticky Header
          </header>
        );
      };
