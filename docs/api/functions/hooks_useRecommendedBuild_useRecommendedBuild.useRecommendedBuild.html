      • applyRecommendedBuild: (build: RecommendedBuild) => void

        Overwrites the current grid state with the layout defined in a recommended build.

        Remarks

        Performs validation on the build object, maps module IDs to full module data, and initiates a scroll-into-view on mobile if needed.

        See

        • isValidRecommendedBuild
        • createGrid
        • createEmptyCell

        Example

        applyRecommendedBuild(myBuildObject);
        // Side-effect: GridStore.grid is updated and mobile screen scrolls to top.

      Remarks

      This hook maps technology and module identifiers from the recommended build layout to their actual data properties in the techTree. It handles automatic scrolling on mobile devices and updates the global GridStore.

      See

      • RecommendedBuild for the build configuration schema.
      • TechTree for the underlying tech data structure.
      • useGridStore for state persistence.
      • useBreakpoint for responsive behavior.
      • useScrollGridIntoView for mobile UX.
      • isValidRecommendedBuild for runtime validation.
