# React Best Practices, Anti-Patterns & Code Smells

**Project:** `nms_optimizer-web` (v7.3.0)
**Reviewer:** Amp (Vercel React Best Practices + manual audit)
**Date:** 2026-05-20
**Scope:** Application code under `src/`. Tests, scripts, generated assets excluded.

The codebase is in solid shape overall — modern React 19 + Zustand + Suspense + lazy
splitting, with a deliberate "colocated hook" pattern that keeps render leaves cheap.
The findings below are improvements, not breakages, and are roughly ordered by impact.

---

## 1. High-impact issues

### 1.1 Excessive "backward-compatibility facade hooks" in `uiStore.ts`

**File:** [src/store/ui/uiStore.ts](file:///home/jbelew/projects/nms_optimizer-web/src/store/ui/uiStore.ts#L452-L740)

Six wrapper hooks (`useShakeStore`, `useThemeStore`, `useTechTreeLoadingStore`,
`useModuleSelectionDialogStore`, `useA11yStore`, `useErrorStore`, `useSessionStore`,
`useOptimizeStore`) are built on top of `useUiStore` using `Object.assign(fn, {getState, setState, subscribe})`.

Each facade:

- Constructs a brand new `subState` object on every store change inside `useShallow`,
  which works correctly (`useShallow` does shallow compare) but adds an indirection
  layer that defeats static tooling and increases bundle size for what is essentially
  legacy compatibility shim code.
- Implements bespoke `setState` mapping logic that translates renamed fields
  (`grid_fixed → gridFixedCount`, `isLoading → isTechTreeLoading`, etc.).
  This duplicates the store schema in eight separate places and is brittle — adding
  a field to `UiStore` requires touching multiple facades.
- Several (`useSessionStore`, `useModuleSelectionDialogStore`) also re-export setter
  functions through the subState. Those are stable references from Zustand, so they
  pass shallow equality — but it's an easy footgun if someone adds derived data.

**Recommendation:** Treat the consolidation as the migration target. Replace the
facades with direct `useUiStore((s) => s.x)` selectors at call sites (one mechanical
pass) and delete the wrappers. The current structure is a half-finished refactor.

### 1.2 `dangerouslySetInnerHTML` for i18n strings

**Files:**
- [src/components/ModuleSelectionDialog/ModuleSelectionDialog.tsx](file:///home/jbelew/projects/nms_optimizer-web/src/components/ModuleSelectionDialog/ModuleSelectionDialog.tsx#L226-L286) (4 occurrences)
- [src/components/InstallPrompt/InstallPrompt.tsx](file:///home/jbelew/projects/nms_optimizer-web/src/components/InstallPrompt/InstallPrompt.tsx#L67)

Translation strings are injected with `dangerouslySetInnerHTML={{ __html: t(...) }}`.
Crowdin contributions are user-supplied content; even with reviewer gating, a stray
`<img onerror>` or `<script>` in a PR could ship XSS.

**Recommendation:** Use `<Trans>` from `react-i18next` (already a dependency) for
rich-text translations, or sanitize via DOMPurify before injecting. The pre-rendered
markdown case in
[PrerenderedMarkdownRenderer.tsx](file:///home/jbelew/projects/nms_optimizer-web/src/components/AppDialog/Markdown/PrerenderedMarkdownRenderer.tsx#L97)
is acceptable because the HTML is build-time generated by SSG.

### 1.3 `useState` initialiser is a non-lazy expression

**File:** [src/App.tsx](file:///home/jbelew/projects/nms_optimizer-web/src/App.tsx#L100)

```tsx
const [showWelcome, setShowWelcome] = useState(!userVisited && !activeDialog && !isBot());
```

`isBot()` parses `navigator.userAgent` on every render even though React only uses the
initialiser on mount. Wrap in a lazy initialiser:

```tsx
const [showWelcome, setShowWelcome] = useState(
  () => !userVisited && !activeDialog && !isBot()
);
```

Same pattern shows up cleanly in
[dialogContext.tsx](file:///home/jbelew/projects/nms_optimizer-web/src/context/dialogContext.tsx#L55-L63) — those are correctly lazy.

(Rule: `rerender-lazy-state-init`.)

### 1.4 Per-render IIFE in `BonusStatusIcon`

**File:** [src/components/TechTreeRow/TechTreeRow.tsx](file:///home/jbelew/projects/nms_optimizer-web/src/components/TechTreeRow/TechTreeRow.tsx#L65-L93)

```tsx
const contentData = (() => {
  if (!status) return null;
  ...
})();
```

Allocates a new object on every render. Since this row is rendered for every tech in
the tree and `useTechBonusStore()` returns the entire store (see 2.1), this multiplies
the work. Inline the switch into normal code or memoise with `useMemo`.

(Rule: `rerender-memo` + `rendering-hoist-jsx`.)

### 1.5 `useTechBonusStore()` subscribes to the entire store

**File:** [src/components/TechTreeRow/TechTreeRow.tsx](file:///home/jbelew/projects/nms_optimizer-web/src/components/TechTreeRow/TechTreeRow.tsx#L62)

```tsx
const { getBonusStatus } = useTechBonusStore();
```

Without a selector, every `TechTreeRow` re-renders on any tech-bonus state change —
including unrelated techs. Use:

```tsx
const getBonusStatus = useTechBonusStore((s) => s.getBonusStatus);
```

Then `getBonusStatus(tech)` reads the latest value when called.

(Rule: `rerender-defer-reads`.)

### 1.6 `ModuleGroup` rebuilds the dependency map on every render

**File:** [src/components/ModuleSelectionDialog/ModuleSelectionDialog.tsx](file:///home/jbelew/projects/nms_optimizer-web/src/components/ModuleSelectionDialog/ModuleSelectionDialog.tsx#L157-L196)

`dependencyMap` and the `rankToModuleMap` walk all modules twice on every render of
every group. The dialog re-renders on each checkbox toggle, so this is O(groups × modules)
each time. Memoize on `[modules]`:

```tsx
const dependencyMap = useMemo(() => {
  if (!["atlantid", "cosmetic", "reactor", "upgrade"].includes(groupName)) {
    return new Map<string, string>();
  }
  // ... build map
}, [groupName, modules]);
```

The two `modules.forEach` passes can also be combined into one (Rule:
`js-combine-iterations`).

## 2. Re-render / data-flow smells

### 2.1 Inline `scrollOptions` object in `useOptimize`

**File:** [src/hooks/useOptimize/useOptimize.tsx](file:///home/jbelew/projects/nms_optimizer-web/src/hooks/useOptimize/useOptimize.tsx#L113)

```tsx
const scrollOptions = { skipOnLargeScreens: true };
const { gridContainerRef, scrollIntoView } = useScrollGridIntoView(scrollOptions);
```

New identity each render. If `useScrollGridIntoView` ever puts this in a dep array it
will loop. Move it to a module-level constant.

### 2.2 Unused import causing dead identifier

**File:** [src/hooks/useOptimize/useOptimize.tsx](file:///home/jbelew/projects/nms_optimizer-web/src/hooks/useOptimize/useOptimize.tsx#L12)

```ts
import { useThemeStore as _useThemeStore, useOptimizeStore } from "@/store/ui/uiStore";
```

`_useThemeStore` is imported and never used. Remove (and run `knip`).

### 2.3 Many independent selectors instead of `useShallow`

**File:** [src/hooks/useOptimize/useOptimize.tsx](file:///home/jbelew/projects/nms_optimizer-web/src/hooks/useOptimize/useOptimize.tsx#L88-L98)

Seven separate `useOptimizeStore` calls. Each is a separate subscription. Coalesce
into one `useShallow` selector for clarity and minor perf:

```tsx
const { setShowError, setPatternNoFitTech, status, setSolving, setProgressPercent } =
  useOptimizeStore(useShallow((s) => ({...})));
```

This also avoids needing the `s.status.type === "warning" ? s.status.tech : null`
derivation to be re-computed on every store change.

### 2.4 `useGridCellInteraction` returns brand-new handlers each render

**File:** [src/components/GridCell/useGridCellInteraction.ts](file:///home/jbelew/projects/nms_optimizer-web/src/components/GridCell/useGridCellInteraction.ts)

`handleClick`, `handleTouchStart`, etc. are recreated each render — and then attached
to a `React.memo`-wrapped `GridCell`. The memo helps because the *props* (rowIndex,
columnIndex) don't change, but the new handler identities defeat any child
optimisation if they were ever forwarded further.

Stash handlers in a ref using the `useLatest`/`useEvent`-style pattern already present
in [src/hooks/useLatest/useLatest.ts](file:///home/jbelew/projects/nms_optimizer-web/src/hooks/useLatest/useLatest.ts), or wrap in `useCallback`
with `[rowIndex, columnIndex, isSharedGrid, cell]` deps.

(Rule: `advanced-event-handler-refs`.)

### 2.5 `DialogProvider` does string work on every render

**File:** [src/context/dialogContext.tsx](file:///home/jbelew/projects/nms_optimizer-web/src/context/dialogContext.tsx#L65-L73)

`location.pathname.split("/").filter(Boolean)` plus the `getSupportedLanguages(i18n)`
call run unmemoized. The supported-languages call may be cheap, but combined with
the surrounding `activeDialog` derivation it adds up on every route transition.
Memoize:

```tsx
const activeDialog = useMemo(() => { ... }, [location.pathname, i18n.language]);
```

This also lets the `useMemo(value, [...])` below have a more stable identity.

## 3. Architectural / organisational smells

### 3.1 Many context-providers + Zustand stores

The app uses both React context (`DialogProvider`, `TooltipContext`, `RouteContext`,
`TechTreeProvider`, `TechTreeRowProvider`, `ModuleSelectionProvider`) **and** four
Zustand stores. Some of the providers exist only to thread one or two values into a
subtree.

Pattern is fine in isolation, but the project's
[useMainAppContext.ts](file:///home/jbelew/projects/nms_optimizer-web/src/components/MainAppContent/useMainAppContext.ts)
splits a single context into four hooks — a sign the context value is doing too much
and is causing wide re-renders. Either:

- Promote stable values to Zustand (where selectors prevent re-renders), or
- Split the provider into separate, narrow contexts.

### 3.2 Dialog rendering pyramid in `AppContent`

**File:** [src/App.tsx](file:///home/jbelew/projects/nms_optimizer-web/src/App.tsx#L140-L189)

Five separate `<Suspense fallback={null}>` blocks gating mutually-exclusive dialogs,
each driven by `activeDialog === "..."`. Consider:

```tsx
{activeDialog && (
  <Suspense fallback={null}>
    <RoutedDialogRouter dialog={activeDialog} />
  </Suspense>
)}
```

Single suspense boundary, easier to instrument with analytics, easier to add a new
dialog without growing `App.tsx`. The `isDockerBuild` gating moves into
`RoutedDialogRouter`.

### 3.3 `void` promises with no error handling

**File:** [src/App.tsx](file:///home/jbelew/projects/nms_optimizer-web/src/App.tsx#L242)

```tsx
if (platform) {
  void fetchTechTree(platform);
}
```

A rejected promise will become an unhandled rejection. Add `.catch(Logger.warn)` or
similar. Same pattern likely exists elsewhere — worth a grep.

### 3.4 `if (!mount) return null` instead of `<Activity>`

**File:** [src/components/MainAppContent/MainAppLayout.tsx](file:///home/jbelew/projects/nms_optimizer-web/src/components/MainAppContent/MainAppLayout.tsx#L65)

React 19 ships `<Activity>` for "render now but hide" semantics. Since the project
already uses React 19, swapping `if (!mount) return null` for
`<Activity mode={mount ? 'visible' : 'hidden'}>...` would let React pre-render the
deferred subtree at idle priority and reveal it instantly. Lower priority — only
worth it if FID matters.

(Rule: `rendering-activity`.)

## 5. Things you're doing well

These are worth calling out so they don't get refactored away:

- **Aggressive `lazy()` + `lazyNamed()` splitting** — dialogs, routes, even
  `OfflineBanner`. Good first-paint discipline.
- **Suspense boundaries** are local and scoped — no top-level "freeze everything"
  fallback.
- **Colocated hook pattern** (`useTechTreeRow`, `useGridCellInteraction`,
  `useMainAppLogic`) keeps prop-drilling low and pushes state reads into the leaves
  that actually need them.
- **Discriminated unions in store types** (`OptimizeStatus`) — proper modeling, not
  just nullable bags.
- **Eager waterfall busting** in [App.tsx](file:///home/jbelew/projects/nms_optimizer-web/src/App.tsx#L235-L244) — preloading
  the tech tree the moment we see `?platform=` in the URL.
- **`useLatest`** ref pattern used in `useOptimize` is exactly the
  `advanced-use-latest` Vercel rule — keeps callback identity stable without
  stale-closure bugs.
- **`debouncedStorage`** for grid persistence — avoids the classic Zustand-persist
  perf cliff.
- **`React.memo` on `GridCell` and `TechTreeRow`** — the two most-rendered components.
- **Skip-link in `MainAppLayoutContent`** — accessibility wins.

## 6. Recommended next steps (ordered)

1. **Fix `dangerouslySetInnerHTML` on i18n strings** (security). Replace with `<Trans>`.
2. **Switch `useTechBonusStore()` → selector form** in `BonusStatusIcon` — biggest
   render-perf win for the tech tree.
3. **Memoise `dependencyMap` in `ModuleGroup`** and the `BonusStatusIcon` IIFE.
4. **Lazy-init the `showWelcome` state** in `App.tsx`.
5. **Coalesce `useOptimize` selectors with `useShallow`** and remove the unused
   `_useThemeStore` import.
6. **Plan the deletion of the legacy facade hooks** in `uiStore.ts` — schedule a
   mechanical migration sprint.
7. **Add `.catch(...)` to `void fetchTechTree(platform)`** and grep for similar
   `void promise` patterns.
8. **Audit `React.memo` usage** — confirm parent components aren't passing new
   prop identities (objects, inline handlers) that defeat the memo.

None of these are urgent; the app is well within healthy React-app territory.
